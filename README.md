# OpenChat

Проект для управления чатами.

Это часть Server core.

## Подход к разработке

**Не монорепа** — каждый компонент в своём репозитории.

**Порядок разработки на каждом этапе:**

1. **Server core** — библиотека с серверным API, все методы зашиты внутри (текущий репо)
2. **Server** — бэкенд, использует server core
3. **Client core** — библиотека с клиентским API, дружит с server core
4. **Webapp / TUI** — интерфейс поверх client core

**Суть:** сначала пишем две библиотеки (серверное и клиентское API), потом натягиваем интерфейс.

## Roadmap

### Этап 1: Базовый чат

- [ ] Подключение к серверу, handshake
- [ ] Email-верификация
- [ ] Создание/список чатов
- [ ] Отправка/получение текстовых сообщений
- [ ] История сообщений

### Этап 2: Профили и участники

- [ ] Профиль пользователя
- [ ] Список участников чата
- [ ] Роли (владелец/модератор/участник/наблюдатель)
- [ ] Приглашения в чат

### Этап 3: Шифрование

- [ ] Identity key (keypair на клиенте)
- [ ] Публичные чаты (ключ от сервера)
- [ ] Защищённые чаты (E2E)

### Этап 4: Relay-сеть

- [ ] Доверенные сервера
- [ ] Кросс-аутентификация через challenge
- [ ] Multi-endpoint

### Этап 5: Медиа и звонки

- [ ] Медиа-сообщения (изображения, видео, аудио)
- [ ] WebRTC-звонки

### Этап 6: Дополнительно

- [ ] Голосования
- [ ] Офлайн-доставка (статусы сообщений)
- [ ] MessagePack
